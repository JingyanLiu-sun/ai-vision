## 背景确认
- 站点地图由 `scripts/generate-sitemap-rss.mjs` 生成，会扫描 `src/app/[lang]/**/page.js` 并用正则解析 `PageMeta({...})` 内的 `title/description/canonicalUrl/publishedDate/updatedDate`。
- 这两个算法的路由文件：
  - `src/app/[lang]/algorithms/intervalscheduling/page.js:13-19` 使用 `dict.seo.intervalscheduling.*` 并提供 `canonicalUrl`、时间戳。
  - `src/app/[lang]/algorithms/minimumspanningtree/page.js:13-19` 使用 `dict.seo.minimumspanningtree.*` 并提供 `canonicalUrl`、时间戳。
- 生成器对 `title/description` 的获取使用 `getNestedValue(dict, key)`，因此必须在字典中存在 `seo.intervalscheduling` 与 `seo.minimumspanningtree`。

## 计划步骤
1. 在字典补齐 SEO 键（若缺失）
- `src/app/dictionaries/zh.json` 与 `src/app/dictionaries/en.json` 的 `seo` 下新增：
  - `seo.intervalscheduling: { title, description, keywords }`
  - `seo.minimumspanningtree: { title, description, keywords }`
- 文案与关键词与页面一致，确保生成器解析后可正确填充。

2. 执行站点地图生成
- 运行生成脚本，生成以下文件：
  - `public/sitemap-en.xml`、`public/sitemap-zh.xml`、`public/sitemap.xml`
  - `public/rss.xml`（zh）与 `public/rss-en.xml`

3. 校验收录结果
- 检查 `public/sitemap-en.xml` 与 `public/sitemap-zh.xml` 是否包含：
  - `https://gallery.selfboot.cn/en/algorithms/intervalscheduling`
  - `https://gallery.selfboot.cn/en/algorithms/minimumspanningtree`
  - `https://gallery.selfboot.cn/zh/algorithms/intervalscheduling`
  - `https://gallery.selfboot.cn/zh/algorithms/minimumspanningtree`
- 检查 `lastmod` 与 `priority/changefreq` 是否符合脚本默认值。
- 检查 RSS 是否新增对应条目标题与描述。

4. 如发现解析失败，按以下顺序修正：
- 调整 `PageMeta({...})` 字段位置与格式，使正则能匹配到：`title:`、`description:`、`canonicalUrl:`、`publishedDate:`、`updatedDate:`（参考 `src/app/[lang]/algorithms/intervalscheduling/page.js:12-19` 格式）。
- 若需要，优化生成器正则以兼容更灵活的换行与空格，但保持简单可靠。

## 验收标准
- 两语言的算法页面均出现在 sitemap 与 sitemap 索引中，并在 RSS 中有对应条目。
- `canonicalUrl` 指向正确路由；`lastmod` 来源于 `updatedDate` 字段。
- 不影响其他页面的 sitemap/RSS 生成。
